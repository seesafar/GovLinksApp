SECTION 01 Concept

What is UDSAL Schema?

UDSAL Schema is a unified data model for describing digital services from different countries in a stable, predictable way. It allows thousands of services to be linked through a single access layer without any server-side logic.

Each service is stored as a JSON object with fields for identity, region, category, official URL, call-to-action, and tags used by the global deep search engine inside OneLink.

SECTION 06 Future Extensions

Towards UDSAL Schema 2.0

Once v1.0 is stable across Saudi Arabia, Egypt, China, and global services, UDSAL Schema can be extended with more advanced fields such as:

  • Authentication profile (identity provider, required level)
  • Multi-region routing hints (preferred domain per country)
  • Service relations (depends-on, related-to, replaces)
  • Availability metadata (beta, preview, migration status)
  • Analytics hooks and experiment flags

These extensions will be defined in UDSAL Schema v2.0 while keeping backwards compatibility with the v1.0 core fields described in this document.

SECTION 07 Role inside OneLink

How OneLink Uses the Schema

In the current implementation, all services are loaded from a single JSON file (for example services-data.json) that follows this schema. The OneLink Global Hub and each national portal (SA, EG, CN) use the same data model, filtered by region and tags.

This means the front-end can power:

  • Global deep search
  • Region tabs (Saudi / Egypt / China / Global / All)
  • Category filters and smart chips
  • Future AI-based recommendations
